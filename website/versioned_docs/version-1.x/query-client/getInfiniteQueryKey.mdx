# getInfiniteQueryKey(...)

The method provides a standardized way to generate `QueryKey` the _Infinite Queries_.
See TanStack [_Query Keys ðŸŒ´_](https://tanstack.com/query/latest/docs/framework/react/guides/query-keys)
guide for more information.

```ts
const queryKey = qraft.<service>.<operation>.getInfiniteQueryKey(parameters);
```

### Arguments

1.  `parameters: { path, query, header } | QueryKey | {}`
    - **Required**, OpenAPI request parameters for the query, strictly-typed âœ¨
    - `parameters` will be used to generate the `QueryKey`
    - If operation does not require parameters, you must pass an empty object `{}` for strictness

### Returns

`QueryKey` - a query key for the _Infinite Queries_

### Example

```tsx
const queryKey = qraft.files.getFiles.getInfiniteQueryKey({
  header: { 'x-monite-version': '1.0.0' },
  query: { id__in: ['1', '2'] },
});

expect(queryKey).toEqual([
  {
    method: 'get',
    url: '/files',
    infinite: true // â¬…ï¸Ž this is the only difference from the `getQueryKey` method result
  },
  {
    header: { 'x-monite-version': '1.0.0' },
    query: { id__in: ['1', '2'] }
  }
]);
```
